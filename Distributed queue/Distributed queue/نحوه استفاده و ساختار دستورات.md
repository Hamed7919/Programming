### نحوه استفاده و ساختار دستورات

این ابزار ۱۰ فرمان اصلی دارد که هر کدام زیرمجموعه‌های خودشان را دارند. مهم‌ترین‌ها عبارتند از:

- **`server`**: برای مدیریت سرور (مثلاً `temporal server start-dev`).
    
- **`workflow`**: برای کار با ورک‌فلوها (مثلاً `temporal workflow start` یا `list`).
    
- **`operator`**: برای مدیریت کلاستر.
    

> **طلایی‌ترین نکته:** لازم نیست دستورات را حفظ کنید! کافیست بعد از هر بخش، عبارت **`--help`** را اضافه کنید تا راهنمای کامل آن ظاهر شود. _مثلاً:_ `temporal workflow --help`
