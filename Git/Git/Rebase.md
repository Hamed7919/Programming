**استراتژی‌های ترکیب کد ([[Git merge]] vs Rebase)**


| Rebase                              | Merge                         | ویژگی       |
| ----------------------------------- | ----------------------------- | ----------- |
| خطی (یک خط صاف و تمیز)              | غیرخطی (نمودار شاخه‌دار)      | شکل تاریخچه |
| تاریخچه را بازنویسی می‌کند (تمیزتر) | تاریخچه واقعی را حفظ می‌کند   | اصالت       |
| برای شاخه‌های شخصی و موقت           | برای شاخه‌های اصلی ([[Main]]) | کاربرد      |


> نکته حیاتی در Rebase: اگر کدی را قبلاً [[Git push]] کرده‌اید، آن را Rebase نکنید. اگر مجبور شدید، باید از
```
>  git push --force-with-lease 
```
>  استفاده کنید تا تغییرات دیگران تصادفاً پاک نشود.

___

خلاصه طلایی برای ذخیره:
1. ​قانون اصلی: فقط روی [[git branch]]های شخصی که هنوز کسی به آن‌ها دست نزده [[Rebase]] کن.
2. ​قانون [[Git push]]: اگر کدی عمومی شده، به جای Rebase از [[Git merge]] استفاده کن.
3. ​قانون نجات: اگر اشتباهی Rebase کردی، حتماً از
```
--force-with-lease 
```

 استفاده کن تا کارِ کسی را خراب نکنی.
