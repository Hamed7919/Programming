وقتی کار در شاخه تستی تمام شد، نوبت به برگرداندن آن به شاخه اصلی می‌رسد:

1. ابتدا به شاخه مقصد (مثلاً [[Main]]) بروید: 
2. git checkout main
3. دستور مرج را بزنید: git [[merge]] develop
* نتیجه: تمام تغییرات شاخه develop به main منتقل می‌شود
___
بله، دقیقاً! اگر شاخه test تو جلوتر از main باشه (یعنی کامیت‌های جدیدتری داری که می‌خوای main رو به آخرین کامیت test برسونی)، این کار رو با یکی از دو روش زیر انجام می‌دی:

### روش پیشنهادی و تمیز (با merge):
1. اول روی شاخه main برو:
   ```bash
   git checkout main
   ```

2. تغییرات ریموت رو بگیر (اگر لازم باشه):
   ```bash
   [[Git fetch]]
   ```

3. شاخه test رو به main merge کن:
   ```bash
   git merge test
   ```

   - نتیجه: main به آخرین کامیت test می‌رسه و یک کامیت merge ساخته می‌شه (تاریخچه حفظ می‌شه).

3. بعد [[Git push]] کن به ریموت:
   ```bash
   git push origin main
   ```


### روش جایگزین (اگر تاریخچه خطی بخوای – با rebase):
روی main باش و:
```bash
git rebase test```

- این کار کامیت‌های main رو روی انتهای test می‌چینه (تاریخچه خطی می‌شه).
- بعد push (اگر لازم باشه با --force-with-lease).

### خلاصه‌ترین:
برای آوردن main به آخرین کامیت test → روی [[Main]] باش و git merge test بزن.  
ساده، امن و استاندارد!
