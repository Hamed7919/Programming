**۱. بخش «خیر» (تنظیمات کلی سیستم):**

​کلیدهای SSH که در مرحله اول ساختی ([[id_ed25519]]) و آن را در سایت گیت‌هاب کپی کردی، مربوط به کل سیستم و لپ‌تاپ توست. یعنی از این به بعد گیت‌هاب، این کامپیوتر را به عنوان یک دستگاه مطمئن می‌شناسد.
​تو لازم نیست برای هر پروژه دوباره کلید بسازی یا دوباره به تنظیمات گیت‌هاب بروی.
**​۲. بخش «بله» (تنظیمات هر پروژه):**

​هر پروژه‌ای که در سیستم خودت داری، یک فایل تنظیمات داخلی دارد. گیت باید بداند که در آن پروژه خاص، قرار است از چه روشی (HTTPS یا SSH) با سرور حرف بزند.
​پس اگر پروژه جدیدی بسازی، فقط کافی است موقع تعریف ریموت، آدرس SSH (که با @git شروع می‌شود) را به آن بدهی.
​برای پروژه‌های قدیمی که قبلاً با [[HTTPS]] بودند هم باید مثل همین پروژه‌ای که با هم انجام دادیم، یک‌بار دستور set-url را بزنی تا مسیرشان به سمت SSH کج شود.

___

بسیار عالی، نکته اصلی اینجاست که از این به بعد باید به جای لینک‌های [[HTTPS]]، همیشه از لینک‌های SSH استفاده کنی.
در ادامه، دستور کلی برای دو حالت اصلی را برایت می‌نویسم:
۱. اگر می‌خواهی یک پروژه جدید را از گیت‌هاب بگیری ([[۴. کپی برداری (Clone)]])
وقتی در صفحه پروژه در گیت‌هاب هستی، روی دکمه سبز رنگ Code بزن و حتماً تب SSH را انتخاب کن. دستور آن همیشه این شکلی است:
```
git clone git@github.com:USERNAME/REPOSITORY_NAME.git

```
**در این حالت، گیت خودش می‌فهمد که باید از کلید SSH سیستم تو استفاده کند و دیگر هیچ‌وقت از تو یوزرنیم نمی‌خواهد.**
۲. اگر پروژه را در لوکال داری و می‌خواهی به ریموت وصل کنی ([[۲. شروع از صفر (Local to Remote)]])
اگر پروژه را با [[Git init]] ساخته‌ای و حالا می‌خواهی به یک مخزنِ خالی در گیت‌هاب وصلش کنی، دستور کلی این است:
```
git remote add origin git@github.com:USERNAME/REPOSITORY_NAME.git
```

(یادت باشد: همیشه آدرسی که با git@github.com شروع می‌شود را کپی کن).
۳. یک ترفند سریع برای تغییر پروژه‌های قدیمی
اگر پروژه‌های زیادی داری که قبلاً با HTTPS تنظیم شده‌اند و نمی‌خواهی آدرس تک‌تک آن‌ها را حفظ کنی، از این قالب استفاده کن:
```
git remote set-url origin git@github.com:Hamed7919/PROJECT_NAME.git

```

خلاصه تفاوت در ظاهر آدرس‌ها:
 * ❌ HTTPS (بد): https://github.com/user/repo.git (یوزرنیم و پسورد می‌خواهد)
 * ✅ SSH (خوب): git@github.com:user/repo.git (با کلیدِ تو کار می‌کند)
> پیشنهاد: همین الان برو سراغ یکی از پروژه‌هایت و با دستور git remote -v چک کن ببین آدرسش با کدام شروع می‌شود. اگر با https بود، با دستوری که بالا گفتم به git@ تغییرش بده تا قدرت SSH را حس کنی!
