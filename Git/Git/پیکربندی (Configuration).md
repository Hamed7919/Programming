

قبل از اولین [[git commit]]، باید هویت خود را مشخص کنید تا در تاریخچه مشخص باشد چه کسی تغییر را ایجاد کرده:

```bash
git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"
```


___

تنظیمات گیت مشخص می‌کند که شما چه کسی هستید، از چه ابزارهایی استفاده می‌کنید و رفتار پیش‌فرض گیت چگونه باشد.

### ۱. تعیین هویت (User Identity)

قبل از هر کامیت، باید مشخص کنید که مسئولیت تغییرات با کیست. این اطلاعات در تاریخچه (Log) نمایش داده می‌شود.

* نام کاربری: 
```
* git config --global user.name "Your Name"
```
* ایمیل:
```
* git config --global user.email "you@example.com"

```
### ۲. سطوح تنظیمات (Configuration Levels)

گیت تنظیمات را در سه لایه مختلف می‌خواند. لایه‌های پایین‌تر (جزئی‌تر) تنظیمات لایه‌های بالاتر (کلی‌تر) را لغو (Override) می‌کنند:

| سطح (Level) | دستور    | محدوده اثر                              | اولویت      |
| ----------- | -------- | --------------------------------------- | ----------- |
| [[System]]  | --system | تمام کاربران سیستم و تمام ریپازیتوری‌ها | ۳ (کمترین)  |
| [[Global]]  | --global | تمام پروژه‌های کاربر فعلی               | ۲           |
| [[Local]]   | --local  | فقط ریپازیتوری فعلی (پیش‌فرض)           | ۱ (بیشترین) |


___


### ۳. شخصی‌سازی‌های کاربردی

یکی از مهم‌ترین تنظیمات اولیه، تغییر نام شاخه پیش‌فرض از [[master]] به [[main]] است تا با استانداردهای جدید [[GitHub]] هماهنگ باشید:
* تغییر شاخه پیش‌فرض:
```
* git config --global init.defaultBranch main
```

### ۴. بررسی تنظیمات

برای اینکه بفهمید در حال حاضر چه تنظیماتی روی سیستم شما فعال است:

* لیست کامل:
```
* git config --list
```
* مشاهده یک مورد خاص:
```
* git config user.name
```
