در گیت، ما هیچ‌وقت مستقیماً روی کد اصلی (شاخه [[Main]]) که در حال اجراست ریسک نمی‌کنیم.

* ایجاد کپی مجازی: وقتی برنچ می‌سازید، در واقع یک مسیر موازی برای توسعه ایجاد می‌کنید.
* امنیت: اگر در برنچ تستی کد خراب شد، کد اصلی در [[main]] سالم می‌ماند.

___

### ۲. کار با برنچ‌ها (دستورات پایه)

* ایجاد برنچ جدید: 
* git branch <name>
* رفتن به یک برنچ:
* [[Git checkout]] <name>
* میانبر (ساخت و ورود همزمان):
* git checkout -b <name>
* برنچ گرفتن از یک زمان خاص: اگر آدرس یک کامیت قدیمی را داشته باشید، می‌توانید از همان نقطه زمان را به عقب برگردانید و برنچ بسازید:
git checkout -b <new-branch-name> <commit-id>

___


> ---
>  **نکته مهم:** شما فقط می‌توانید از [[git commit]] برنچ بگیرید. اگر تغییراتی دارید که هنوز کامیت نشده‌اند ([[Staged]] هستند)، ابتدا باید آن‌ها را کامیت کنید تا بتوانید برنچ تمیزی بسازید.



___

* مشاهده تغییرات ذخیره نشده: [[Git diff]] (تفاوت ورکینگ دایرکتوری با استیج).
* مقایسه دو برنچ: git diff branch1 branch2
* تحلیل خروجی:
* --- و +++: فایل‌های مورد مقایسه.
* -: خطوطی که حذف شده‌اند (با رنگ قرمز).
* +: خطوطی که اضافه شده‌اند (با رنگ سبز).
