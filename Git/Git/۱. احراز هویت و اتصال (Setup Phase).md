مسیر A: احراز هویت با [[SSH]] (پیشنهادی)
گام 1 — بررسی کلید موجود
```
 ls ~/.ssh
```

اگر [[id_ed25519]] یا [[id_rsa]] دیدی → برو گام 3
اگر نبود → گام 2

___

==گام 2 — ساخت SSH Key==

`ssh-keygen -t ed25519 -C "your_email@example.com"`

Enter برای مسیر پیش‌فرض

Passphrase اختیاری (امن‌تر)

___

گام 3 — حالا باید کلیدی که ساخته شده را کپی کنی تا به گیت‌هاب بدهی.

```
`cat ~/.ssh/id_ed25519.pub

```

(متنی که چاپ می‌شود و با ssh-ed25519 شروع می‌شود را کپی کن).
___

گام 4 — کپی کلید عمومی

`cat ~/.ssh/id_ed25519.pub`

⬅️ اینو ببر داخل [[GitHub]] / GitLab → [[SSH]] Keys

___

گام 5 — تست اتصال

`ssh -T git@github.com`

✅ اگر پیام خوش‌آمد دیدی → احراز هویت کامله
___
==نکته مهم اگر از توان استفاده می‌کردی و الان میخوای بری سمت== ==ssh==  ==:==

 **تغییر آدرس ریموت به SSH**
​حالا باید به پروژه‌ات بگویی که دیگر از آدرس HTTPS استفاده نکند و از آدرس SSH استفاده کند. در ترمینالِ پروژه‌ات این دستور را بزن:
```
git remote set-url origin git@github.com:USERNAME/REPOSITORY.git

```

(به جای USERNAME نام کاربری خودت و به جای REPOSITORY اسم پروژه‌ات را بنویس. این آدرس دقیق را می‌توانی در دکمه سبز رنگ Code در گیت‌هاب، بخش SSH ببینی).
___
